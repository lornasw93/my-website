DevOp Stats

Prerequisites

  • Token key generated from Azure DevOps

I started up this project for my own learning with the purpose being to get an overview of useful everyday stats from DevOps. The expected outcome is to have 3 projects working together (frontend, backend and identity server). At a previous job, I found that the same questions were always asked:

  • Who broke the build?
  • When was the last release?
  • Balancing out work items
  • Generally people not being aware of what other people were up to project wise
  • Can someone please send over the UAT URL?
  • Which server is Website A hosted on?

I wanted to find out basic info quickly rather than going through several pages to find out what I wanted. I.e. Why did the last build fail? To summarise, my goal was to condense common queries into a simple dashboard where then, as a typical developer, I would benefit from the following:

  • Efficiency
  • Less clicks
  • Save time by asking same things
  • (Chart) trends
  • Visualisation
